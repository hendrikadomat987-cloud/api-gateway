# Requests Service

Multi-tenant CRUD service for managing customer support requests (callback, support, quote, info).

---

## Architecture

```
Client → API Gateway (JWT auth + tenant injection) → n8n Webhook → PostgreSQL RPC (RLS)
```

---

## Data Model

| Field       | Type        | Required | Notes                                                       |
|------------|-------------|----------|-------------------------------------------------------------|
| id         | UUID        | auto     | Generated by DB                                             |
| tenant_id  | UUID        | JWT only | Never from client — injected by gateway                     |
| customer_id| UUID        | yes      | Must be a valid UUID                                        |
| type       | text        | yes      | `callback` \| `support` \| `quote` \| `info`               |
| status     | text        | no       | `pending` (default) \| `in_progress` \| `resolved` \| `closed` |
| notes      | text        | no       | Free-form text                                              |
| created_at | timestamptz | auto     | Set on insert                                               |
| updated_at | timestamptz | auto     | Updated on every change                                     |

## API Endpoints

| Method | Path                     | Operation            |
|--------|--------------------------|----------------------|
| POST   | /api/v1/requests         | Create request       |
| GET    | /api/v1/requests         | List all requests    |
| GET    | /api/v1/requests/:id     | Get request by ID    |
| PUT    | /api/v1/requests/:id     | Partial update       |
| DELETE | /api/v1/requests/:id     | Delete (idempotent)  |

### Request / Response Examples

**POST /api/v1/requests**
```json
{
  "customer_id": "uuid",
  "type": "support",
  "status": "pending",
  "notes": "Customer needs help with billing"
}

// Response 200
{ "success": true, "data": { "id": "uuid", "tenant_id": "uuid" } }
```

**GET /api/v1/requests/:id**
```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "tenant_id": "uuid",
    "customer_id": "uuid",
    "type": "support",
    "status": "pending",
    "notes": "...",
    "created_at": "2026-03-26T00:00:00Z",
    "updated_at": "2026-03-26T00:00:00Z"
  }
}
```

**PUT /api/v1/requests/:id** — at least one field required
```json
{ "status": "resolved" }
```

**Error format**
```json
{ "success": false, "error": { "code": "VALIDATION_ERROR", "message": "..." } }
```

## Security

- `tenant_id` is **never** accepted from the client body
- Gateway extracts `organization_id` (or `tenant_id`) from the verified JWT and injects it before forwarding
- PostgreSQL RLS + FORCE RLS enforces tenant isolation at the DB layer
- All IDs validated as UUIDs before forwarding to n8n
- `type` and `status` validated against allowed enum values

|------------------------------------------------------|----------------|

## Deployment Phases

### Phase 1 — Generate
All artifacts in this folder are generated and ready.

### Phase 2 — Apply / Deploy

Execute in this exact order:

1. **Apply SQL to Supabase**
   ```sql
   \i requests.sql
   ```

2. **Import and activate n8n workflows**
   - Open n8n → Workflows → Import from file
   - Import each `_requests_*.json` file
   - Activate all 5 workflows
   - Verify webhook URLs are live

3. **Deploy gateway changes**
   - Ensure `requests` entry exists in `api-gateway/config.js → services`
   - Ensure POST and PUT validation blocks for `requests` are in `api-gateway/src/routes/apiRouter.js`
   - Restart / redeploy the gateway

### Phase 3 — Test

> **Run tests ONLY after all Phase 2 steps are complete.**

```bash
# From project root — using test-engine/tests/ copies (canonical)
node test-engine/run-tests.js test-engine/tests/requests/requests.crud.test.js
node test-engine/run-tests.js test-engine/tests/requests/requests.gateway-security.test.js
node test-engine/run-tests.js test-engine/tests/requests/requests.rls.test.js
```

Prerequisites before running:
- `requests.sql` applied to Supabase ✓
- All 5 n8n workflows imported and active ✓
- Gateway restarted with requests service registered ✓
- `VALID_CUSTOMER_ID` in test files updated to a real customer UUID for your test tenant ✓
