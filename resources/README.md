# Resources Service

Multi-tenant CRUD service for managing digital resources (documents, templates, scripts, FAQs) used by the AI voice agent.

---

## Architecture

```
Client → API Gateway (JWT auth + tenant injection) → n8n Webhook → PostgreSQL RPC (RLS)
```

---

## Data Model

| Field     | Type        | Required | Notes                                              |
|----------|-------------|----------|----------------------------------------------------|
| id       | UUID        | auto     | Generated by DB                                    |
| tenant_id| UUID        | JWT only | Never from client — injected by gateway            |
| name     | text        | yes      | Resource name / title                              |
| type     | text        | yes      | `document` \| `template` \| `script` \| `faq`     |
| content  | text        | no       | Resource body / content                            |
| status   | text        | no       | `active` (default) \| `draft` \| `archived`        |
| created_at | timestamptz | auto   | Set on insert                                      |
| updated_at | timestamptz | auto   | Updated on every change                            |

---

## API Endpoints

| Method | Path                      | Operation            |
|--------|---------------------------|----------------------|
| POST   | /api/v1/resources         | Create resource      |
| GET    | /api/v1/resources         | List all resources   |
| GET    | /api/v1/resources/:id     | Get resource by ID   |
| PUT    | /api/v1/resources/:id     | Partial update       |
| DELETE | /api/v1/resources/:id     | Delete (idempotent)  |

### Request / Response Examples

**POST /api/v1/resources**
```json
{
  "name": "Customer FAQ v1",
  "type": "faq",
  "content": "What are your opening hours?...",
  "status": "active"
}

// Response 200
{ "success": true, "data": { "id": "uuid", "tenant_id": "uuid" } }
```

**GET /api/v1/resources/:id**
```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "tenant_id": "uuid",
    "name": "Customer FAQ v1",
    "type": "faq",
    "content": "...",
    "status": "active",
    "created_at": "2026-03-26T00:00:00Z",
    "updated_at": "2026-03-26T00:00:00Z"
  }
}
```

**PUT /api/v1/resources/:id** — at least one field required
```json
{ "status": "archived" }
```

**Error format**
```json
{ "success": false, "error": { "code": "VALIDATION_ERROR", "message": "..." } }
```

## Security

- `tenant_id` is **never** accepted from the client body
- Gateway extracts `organization_id` (or `tenant_id`) from the verified JWT and injects it before forwarding
- PostgreSQL RLS + FORCE RLS enforces tenant isolation at the DB layer
- All IDs validated as UUIDs before forwarding to n8n
- `type` and `status` validated against allowed enum values

| Function                                                   | Description    |
|-----------------------------------------------------------|----------------|
| `resource_create(tenant_id, name, type, content, status)` | Insert         |
| `resource_list(tenant_id)`                                 | List all       |
| `resource_get_by_id(tenant_id, id)`                       | Get one        |

## Gateway Integration

See `resources.gateway.patch.js` for:
- Service registry entry (`config.js`)
- Validation blocks (`apiRouter.js`)

Add to `api-gateway/config.js → services`:
```js
resources: {
  POST:      'resources/create',
  GET:       'resources/list',
  GET_ID:    'resources/get',
  PUT_ID:    'resources/update',
  DELETE_ID: 'resources/delete',
},
```

## Deployment Phases

### Phase 1 — Generate
All artifacts in this folder are generated and ready.

### Phase 2 — Apply / Deploy

Execute in this exact order:

1. **Apply SQL to Supabase**
   ```sql
   \i resources.sql
   ```

2. **Import and activate n8n workflows**
   - Open n8n → Workflows → Import from file
   - Import each `_resources_*.json` file
   - Activate all 5 workflows
   - Verify webhook URLs are live

3. **Deploy gateway changes**
   - Add `resources` entry to `api-gateway/config.js → services` (see above)
   - Add POST and PUT validation blocks for `resources` to `api-gateway/src/routes/apiRouter.js`
   - Restart / redeploy the gateway

### Phase 3 — Test

> **Run tests ONLY after all Phase 2 steps are complete.**

```bash
# From project root — using test-engine/tests/ copies (canonical)
node test-engine/run-tests.js test-engine/tests/resources/resources.crud.test.js
node test-engine/run-tests.js test-engine/tests/resources/resources.gateway-security.test.js
node test-engine/run-tests.js test-engine/tests/resources/resources.rls.test.js
```

Prerequisites before running:
- `resources.sql` applied to Supabase ✓
- All 5 n8n workflows imported and active ✓
- Gateway restarted with resources service registered ✓
